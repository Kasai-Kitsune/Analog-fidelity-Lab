AUTHENTICITY RESEARCH — Why N64 Looks Like N64

Hardware Limitations Enforced

  • 320×240 Native: Reality Coprocessor (RCP) rendered most games at 320×240 @ ~15.7kHz. 640×480 existed but cut fillrate in half — only Perfect Dark/RE2 used it. Source: RCP programming manual.
  • RGBA5551 Framebuffer: RDRAM stored 16-bit color (5 bits/channel + 1 alpha). True 24-bit mode required 2× bandwidth and was almost never used commercially.
  • 4KB TMEM: Texture memory was 4 kilobytes total. Forces 32×32 RGBA or 64×64 CI4 textures. We simulate by downscaling before filtering.
  • 3-Point Bilinear: RDP does NOT do 4-tap bilinear. It uses triangular interpolation (GLideN64: f.x+f.y < 1.0). Causes signature soft diagonals and texture seams.
  • VI Blur + AA: Video Interface horizontally resamples 320→640 and applies 5-tap filter. Combined with coverage-based anti-aliasing creates the famous N64 softness.
  • 2×2 Ordered Dither: VI_CONTROL_REG bit 16 enables dither filter to hide 16-bit banding. We use Bayer matrix [[0,2],[3,1]].

Pipeline (WebGL2)

  • Pass 1: Nearest-neighbor downscale to N64 res (no smoothing). Simulates RDP pixel fill.
  • Pass 2 Shader: 1) 3-point sample, 2) VI blur/deblur, 3) coverage AA, 4) CRT bleed, 5) dither, 6) quantize to 5-bit, 7) gamma. All per hardware.
  • This is not a filter — it's a cycle-accurate approximation of RDP → VI → DAC.
